# 四旋翼飞行器的部分硬件组成

## 飞控（FC）

飞行控制器（FC）是自动驾驶仪的硬件，PX4固件会上传到该硬件上。Pixhawk 是 PX4 的参考硬件平台，并在 [NuttX](https://nuttx.apache.org/) 操作系统上运行 PX4。

## 传感器

基于PX4的系统利用传感器来估计车辆状态，这对于稳定车辆和实现自主控制至关重要。车辆状态信息包括：**位置/高度、航向、速度、空速、姿态、不同方向的旋转速率、电池电量等等**。PX4至少需要**陀螺仪、加速度计、磁力计（指南针）和气压计**来测量上述状态。[Pixhawk系列](https://docs.px4.io/main/en/flight_controller/pixhawk_series)飞控已配备最基本的传感器。

### 必备的传感器

- 加速度计：测量加速度的变化。大多数飞行控制器，都将加速度计作为飞行控制器[惯性运动单元 (IMU)](https://en.wikipedia.org/wiki/Inertial_measurement_unit)的一部分。首次使用前必须对加速度计进行[校准](https://docs.px4.io/main/en/config/accelerometer)。

- 陀螺仪：测量方向。同样是 IMU 的一部分。首次使用前必须对陀螺仪进行[校准](https://docs.px4.io/main/en/config/gyroscope)。

- 磁力计（指南针）：测量航向/方向。飞控基本上都内置了指南针，但不建议将内置指南针用作航向源，因为其性能很差。建议使用外部的**GPS+指南针组合模块**，[并尽可能远离电机/电调电源线安装](https://docs.px4.io/main/en/assembly/mount_gps_compass)——通常安装在底座或机翼上（固定翼飞机）。指南针[校准](https://docs.px4.io/main/en/config/compass)。

- 气压计：测量海拔高度（通过气压）。通常情况下，气压计不需要用户进行任何配置。

### 推荐加装的传感器

- [空速传感器](https://docs.px4.io/main/en/sensor/airspeed) —— 测量空速。强烈推荐用于垂直起降飞机和固定翼飞机，因为它们是唯一能够检测失速的装置。
- [全球导航卫星系统（GNSS/GPS）](https://docs.px4.io/main/en/gps_compass/) —— 测量全球位置。是执行任务以及其他一些自动和手动/辅助模式所必需的。
- [实时定位全球导航卫星系统（RTK GNSS，也称GPS）](https://docs.px4.io/main/en/gps_compass/rtk_gps) —— 精度可达厘米级的全球导航卫星系统。某些配置还允许使用全球导航卫星系统而非磁力计来确定航向。

## 电调和电机

许多PX4无人机使用**无刷电机**，这些电机由飞行控制器通过**电子调速器（ESC）**驱动。ESC接收来自飞行控制器的信号，并以此来控制输送给电机的功率大小。

## 无线电控制系统（RC）

无线电遥控（RC）系统可用于通过手持式遥控器**手动控制**飞行器。需要选择一对相互兼容的发射器/接收器。此外，接收器还必须与 PX4和飞控硬件兼容。

## 数据链路

数据链路是用于将飞行器遥测数据（位置、速度、电池状态等）从飞行控制器传输到地面控制站（以及某些遥控系统），以及将地面控制站的指令传输到飞行器的无线电通道。这些链路通常使用与用于手动遥控飞行器的无线电不同的无线电设备。PX4 使用[MAVLink](https://mavlink.io/en/)协议通过无线电通道传输串行数据。

### MAVLink

MAVLink是一个非常轻量化的消息协议，用于无人机之间的通信以及无人机机载设备之间的通信。

工作流程：

```
1. 打包：
    位置 (x=1, y=2, z=3)
    → 打包成 MAVLink message
2. 发送
	通过USB，串口或者UDP（仿真）进行发送
3. 接收
	QGC / ROS / companion computer 把数据还原成结构化信息
```

## 电力系统

飞控需要稳定的电源，电机、舵机和其他外围设备也需要单独的电源。

在无人机里，PDB 通常指 **Power Distribution Board（电源分配板）**，通常用于为飞行控制器“分流”一个稳压电源，同时还能测量电池电压和飞行器消耗的总电流。PDB 有时会包含飞控 (FC) 的电源模块、电机电调 (ESC) 的电源模块以及用于为舵机供电的电池消除电路 (BEC)。

## 外围设备

主要是相机。

## 机载电脑

飞行控制器运行基于 NuttX 的 PX4 程序，并提供核心飞行和安全代码。而机载电脑通常运行 Linux 系统，因为 Linux 是更适合“通用”软件开发的平台。它们通过高速串口或以太网连接，并通常使用 MAVLink 协议或 uXRCE-DDS 进行通信。

## 串口配置

这边主要对着飞控板 [Holybro Pixhawk 6C](https://docs.px4.io/main/en/flight_controller/pixhawk6c) 来看。我们主要用到的是POWER, TELEM, GPS, PPM/SBUS RC 和 I/O PWM OUT。

```
           ┌──────────────┐
           │   POWER      │ ← 电池/电源输入
           └──────┬───────┘
                  ↓
           ┌──────────────┐
           │   PX4 飞控    │
           └──────┬───────┘
   ┌──────────────┼────────────────┬────────────────┐
   ↓              ↓                ↓                ↓

TELEM        GPS UART         RC (SBUS/PPM)    PWM OUT
(MAVLink)    (GPS协议)        (遥控输入)        (电机输出)

机载电脑      定位模块        遥控器           ESC/电机
```
